# Глоссарий

## Бизнес-термины

| Термин | Описание |
|--------|----------|
| **P&L (Profit & Loss)** | Отчёт о прибылях и убытках. Лист "Факт" в Google Sheets |
| **Маржа (Margin)** | Прибыль проекта = Доход - Расход + Дебиторская задолженность |
| **Маржинальность (Profitability)** | Процент маржи = (Маржа / (Доход + Дебиторка)) * 100% |
| **MarginScore** | Скоринг маржи клиента: 2 (<150K), 5 (<=800K), 7 (<=3M), 10 (>3M) |
| **Дебиторская задолженность (DebtorBalance)** | Столбец "ДЕБ/КРЕ" в P&L. Учитывается в расчёте маржи |
| **Постоянные расходы (ConstantExpenses)** | Расходы компании, не привязанные к конкретному проекту |
| **Чистая прибыль (NetProfit)** | TotalProjectIncome - TotalProjectExpenses - TotalConstantExpenses |
| **BDM Bonus** | Премия менеджеру по развитию бизнеса (часть расходов проекта) |
| **Прямые расходы (DirectExpense)** | Расходы, напрямую связанные с выполнением проекта |
| **Расхождение (Discrepancy)** | Разница между суммами из разных источников данных |
| **Управленческий период** | Месяц/год, к которому относится выплата (может не совпадать с датой) |

## Технические термины

| Термин | Описание |
|--------|----------|
| **TableData** | Унифицированное представление таблицы (независимо от источника) |
| **RowData** | Строка в TableData. Ячейки адресуются как "A1", "B2" и т.д. |
| **DataLocation** | Координаты данных в таблице: год, строки начала/конца, колонки месяцев |
| **Canonical Name** | Каноническое (основное) имя проекта, выбранное ProjectRegistry |
| **Display Name** | Оригинальное имя проекта для отображения пользователю |
| **Normalized Name** | Имя после NameCleaningService.Normalize() - lowercase, без спецсимволов |
| **ProjectRegistry** | Реестр всех проектов. Объединяет варианты имён через Union-Find |
| **IProjectMatcher** | Интерфейс для проверки, известен ли проект в реестре |
| **AlgorithmicMatcher** | Реализация IProjectMatcher через 3-уровневый алгоритм |
| **RegistryBasedMatcher** | Реализация IProjectMatcher через ProjectRegistry |
| **Pipeline Stage** | Этап обработки в конвейере (IPipelineStage) |
| **PipelineContext** | Контейнер данных, передаваемый между стадиями pipeline |

## Источники данных (листы Google Sheets)

| Лист | Парсер | Что содержит |
|------|--------|-------------|
| **Факт** | ProjectPLParser | P&L: проекты, доходы/расходы по месяцам, дебиторка |
| **Доходы** | TotalIncomesParser | Записи о зачислениях: проект, сумма, дата, контрагент |
| **Выплаты** | PaymentsParser | Исходящие платежи: кому, тип, сумма, проект, дата |
| **Итоги месяцев** | ExpensesParser | Расходы по проектам: total, BDM bonus, прямые |
| **Проекты** | ProjectsSheetParser | Метаданные: тип проекта, менеджер, даты старта/конца |

## Нормализация имён (NameCleaningService)

Pipeline нормализации:
1. Удаление ключевых слов "доходы"/"расходы"
2. Нормализация версий (#1.5-2 -> #1)
3. Приведение к lowercase
4. Удаление специальных символов
5. Нормализация пробелов

## Трёхуровневый алгоритм матчинга (ProjectMatchingService)

1. **Точное совпадение** - нормализованные имена идентичны
2. **Substring** - одно имя содержит другое
3. **Без суффиксов** - совпадение после удаления числовых суффиксов (00-01, 00-02)
